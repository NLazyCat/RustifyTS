# Transmute 领域研究总结

## TypeScript 语法研究

### 核心语法
- 变量声明: `let`, `const`, `var`
- 函数: `function`, 箭头函数 `=>`, 默认参数
- 类型注解: `: Type`
- 类型别名: `type T = ...`
- 接口: `interface T { ... }`
- 泛型: `function<T>(x: T): T { return x; }`
- 联合类型: `T | U`
- 交叉类型: `T & U`

### 高级特性
- 可选链: `obj?.prop`
- 空值合并: `a ?? b`
- 可选参数: `function(x?: string) { ... }`
- 剩余参数: `function(...args: T[]) { ... }`
- 解构: `const { a, b } = obj;`
- 模板字符串: `` `hello ${name}` ``
- 类型守卫: `if (typeof x === "string") { ... }`

## Rust 语义研究

### 所有权系统
- **所有权**: 值只能有一个所有者
- **借用**: 可以有多个不可变借用或一个可变借用
- **生命周期**: 引用有效的范围

### 类型系统
- **原始类型**: `i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`, `f32`, `f64`, `bool`, `char`
- **引用类型**: `&T`, `&mut T`
- **智能指针**: `Box<T>`, `Rc<T>`, `Arc<T>`, `RefCell<T>`, `Mutex<T>`
- **集合**: `Vec<T>`, `HashMap<K, V>`, `HashSet<T>`
- **Option**: `Option<T>` 替代 null
- **Result**: `Result<T, E>` 错误处理

## 分析算法研究

### 1. 所有权推导
1. 分析变量定义和使用
   - 遍历语义AST，建立「变量溯源表」（变量ID、定义作用域、类型、是否全局）
   - 标记变量使用场景（函数内/跨函数/闭包捕获）
   - 判定规则：全局变量默认「可能被共享」，函数内变量默认「独占」
2. 检测赋值操作
   - 捕获所有赋值表达式（`a = b`/`a.b = c`），构建「赋值依赖链」
   - 标记赋值类型：初始化赋值/覆盖赋值/字段赋值
   - 判定规则：覆盖赋值→变量可能被修改，字段赋值→目标字段需可变性
3. 确定是否需要移动
   - 分析变量「最后使用位置」：赋值后不再使用→标记「可移动」
   - 检测跨作用域赋值（返回值/闭包捕获）：跨作用域→优先「移动/克隆」
   - 判定规则：非Copy类型+跨作用域→自动插入`move`，Copy类型→直接传递
4. 检测借用需求
   - 统计变量引用次数：引用数>1→共享借用，引用数=1→独占借用
   - 标记借用读写属性：仅读取→`&T`，修改→`&mut T`
   - 判定规则：共享借用禁止同时存在可变借用

### 2. 生命周期推导
1. 分析引用关系
   - 遍历所有引用表达式（`&a`/`a.b`），建立「引用链路表」（引用方ID、被引用方ID、引用起始/结束位置）
   - 标记引用的作用域边界（函数/循环/条件块）
   - 判定规则：引用不能超出被引用方的定义作用域
2. 构建约束图
   - 提取生命周期约束（如：参数引用 ≤ 返回值引用、闭包捕获 ≤ 闭包生命周期）
   - 用「生命周期变量」（`'a`/`'b`）标记每个引用的生命周期
   - 判定规则：约束关系满足「子生命周期 ≤ 父生命周期」
3. 求解约束
   - 用「合一算法」求解生命周期约束，合并等价的生命周期变量
   - 检测生命周期冲突（如：短生命周期引用赋值给长生命周期变量）
   - 判定规则：冲突时自动提升短生命周期（如堆分配）或报错
4. 生成生命周期标注
   - 为函数参数/返回值/结构体字段生成生命周期标注（`fn f<'a>(x: &'a T) -> &'a U`）
   - 简化冗余标注（如省略可推导的生命周期）
   - 判定规则：全局变量默认`'static`，闭包捕获默认绑定到闭包生命周期

### 3. 可变性推导
1. 检测变量修改
   - 捕获所有修改操作（`a = 1`/`a.push(2)`/`a.b = 3`），建立「修改操作表」
   - 标记修改的触发场景（直接赋值/方法调用/索引修改）
   - 判定规则：存在修改操作→变量需`mut`标注
2. 分析方法调用（如 push, insert）
   - 建立「可变方法映射表」（如`push`/`insert`/`remove`→可变方法）
   - 检测调用链：调用可变方法→上游变量需可变性
   - 判定规则：调用可变方法→变量必须标注`mut`，或包装`RefCell`
3. 确定是否需要 `mut`
   - 区分「编译期可变性」（`mut`）和「运行期可变性」（`RefCell`）
   - 全局变量/循环引用变量→优先`RefCell`，局部变量→优先`mut`
   - 判定规则：仅局部修改→`mut`，跨作用域/循环引用修改→`RefCell`

### 4. 引用类型推导
1. 分析引用使用场景
   - 遍历所有引用表达式，标记使用场景（参数传递/返回值/闭包捕获/字段存储）
   - 统计引用的读写属性（仅读/修改）
   - 判定规则：仅读→共享引用`&T`，修改→可变引用`&mut T`
2. 检测引用冲突
   - 检查「同时存在多个可变引用」「可变引用与共享引用共存」等冲突
   - 标记冲突场景（如多线程引用/循环引用修改）
   - 判定规则：冲突时自动替换为`RefCell`（运行期检查）
3. 选择引用类型
   - 无冲突+仅读→`&T`，无冲突+修改→`&mut T`
   - 有冲突+共享→`Rc<RefCell<T>>`，跨线程→`Arc<Mutex<T>>`
   - 判定规则：优先编译期引用（`&`/`&mut`），冲突时降级为运行期可变性

### 5. 闭包捕获推导
1. 分析闭包捕获变量
   - 提取闭包内使用的外部变量，建立「捕获变量表」（变量ID、捕获类型、是否修改）
   - 标记闭包逃逸属性（是否跨函数返回/跨线程）
   - 判定规则：逃逸闭包→捕获变量需所有权，非逃逸→优先借用
2. 检测捕获变量的所有权
   - 分析捕获变量的所有权类型（Unique/Shared/SharedMutable）
   - 标记是否存在循环捕获（闭包捕获自身引用）
   - 判定规则：循环捕获→`Rc::clone`，非循环→优先借用
3. 确定捕获模式
   - 非逃逸+仅读→借用捕获（`|&x|`），非逃逸+修改→可变借用（`|&mut x|`）
   - 逃逸+非Copy→`move`捕获，逃逸+循环引用→`move + Rc::clone`
   - 判定规则：避免悬垂引用，优先最小权限捕获

### 6. 空安全推导
1. 分析空值场景
   - 捕获TS中空值相关操作（`null`/`undefined`/`?.`/`!`），建立「空值操作表」
   - 标记空值风险（如非空断言`!`、可选链`?.`）
   - 判定规则：TS中`T | null`→Rust`Option<T>`
2. 推导空值处理方式
   - 非空断言`!`→`unwrap()`/`expect()`（显式panic）
   - 可选链`?.`→`Option::map`/`and_then`（优雅处理空值）
   - 判定规则：全局/关键逻辑→`expect`（带错误信息），非关键→`unwrap_or_default`
3. 生成空安全代码
   - 自动为TS可选类型生成`Option<T>`，插入空值检查逻辑
   - 避免无脑`unwrap`，优先`match`/`if let`处理空值
   - 判定规则：编译期能确定非空→直接解包，运行期不确定→分支处理

### 7. 类型精度推导
1. 分析TS类型
   - 解析TS模糊类型（`any`/`unknown`/`object`），推导具体类型
   - 标记联合类型（`A | B`）、交叉类型（`A & B`）
   - 判定规则：`any`→`dyn Any`，`unknown`→`dyn std::fmt::Debug`
2. 映射到Rust类型
   - 联合类型→Rust`enum`（带类型标记），交叉类型→Rust`struct`（合并字段）
   - 对象字面量→自动生成匿名`struct`，函数类型→`fn`/`Box<dyn Fn>`
   - 判定规则：静态类型→`struct`/`enum`，动态类型→`trait object`
3. 优化类型精度
   - 消除冗余类型转换，合并重复类型定义
   - 为泛型推导具体类型参数（如`Array<number>`→`Vec<i32>`）
   - 判定规则：优先静态类型（性能），动态类型仅用于无法推导的场景

### 8. 线程安全推导
1. 分析并发场景
   - 检测TS中的异步/多线程操作（`Promise`/`Worker`），建立「并发场景表」
   - 标记共享变量的跨线程访问
   - 判定规则：跨线程访问→需满足`Send + Sync`
2. 推导`Send/Sync`属性
   - 分析类型的组成字段：所有字段实现`Send/Sync`→类型自动实现
   - 标记非线程安全类型（如`Rc`/`RefCell`）
   - 判定规则：非线程安全类型→自动替换为线程安全版本（`Rc→Arc`/`RefCell→Mutex`）
3. 生成线程安全代码
   - 单线程→`Rc<RefCell<T>>`，多线程→`Arc<Mutex<T>>`
   - 跨线程闭包→`move`捕获+`Arc::clone`
   - 判定规则：优先单线程模型，并发场景自动升级为线程安全

### 9. 内存布局推导
1. 分析类型结构
   - 解析TS类/对象的字段类型，计算字段大小和对齐方式
   - 标记嵌套类型（如对象内的对象）、动态大小类型（如数组）
   - 判定规则：固定大小→栈分配，动态大小→堆分配
2. 推导内存分配方式
   - 小类型（≤64字节）+ 短生命周期→栈分配，大类型/长生命周期→堆分配（`Box`）
   - 嵌套循环引用→分散分配+引用计数
   - 判定规则：优先栈分配（性能），堆分配仅用于必要场景
3. 优化内存布局
   - 调整字段顺序减少内存对齐浪费（如把小类型放一起）
   - 为C交互场景生成`#[repr(C)]`标注
   - 判定规则：默认`#[repr(Rust)]`，C交互/内存敏感场景→`#[repr(C)]`

### 10. 错误处理推导
1. 分析TS错误场景
   - 捕获TS中的`throw`/`try/catch`/`Promise.reject`，建立「错误场景表」
   - 标记错误类型（内置错误/自定义错误/网络错误）
   - 判定规则：`throw`→Rust`Result<T, E>`，`catch`→`match`/`?`
2. 推导错误类型
   - 自动生成自定义错误枚举（基于TS抛出的错误信息）
   - 实现`thiserror::Error` trait，添加错误描述
   - 判定规则：不同错误场景→不同枚举变体，统一错误处理接口
3. 生成错误处理代码
   - `throw`→`Err(自定义错误)`，`try/catch`→`match Result`
   - 异步错误（`Promise.reject`）→`Result<(), E>`+`await?`
   - 判定规则：优先`?`简化错误传播，顶层逻辑用`match`处理

### 11. 异步模型推导
1. 分析TS异步操作
   - 捕获TS中的`async/await`/`Promise`/`setTimeout`，建立「异步操作表」
   - 标记异步类型（串行/并行/定时器）
   - 判定规则：`async function`→Rust`async fn`，`Promise`→`Future`
2. 推导异步运行时
   - 检测异步复杂度：简单异步→`std::future`，复杂并发→`tokio`/`smol`
   - 标记需要`Pin`的场景（如自引用异步函数）
   - 判定规则：默认`tokio`
3. 生成异步代码
   - `async/await`→直接映射为Rust`async/await`
   - 自引用异步→自动添加`Pin<Box<dyn Future>>`
   - 判定规则：避免手动`Pin`，优先`async fn`简化代码

### 12. 函数纯度推导
1. 分析函数副作用
   - 检测函数内的副作用操作（修改全局变量/IO操作/调用不纯函数），建立「副作用表」
   - 标记函数的输入输出依赖（仅依赖参数/依赖全局）
   - 判定规则：无副作用+输出仅依赖输入→纯函数
2. 推导函数优化策略
   - 纯函数→标记`#[inline]`，可编译期计算→`const fn`
   - 不纯函数→保留副作用，标记`#[warn(unused_must_use)]`
   - 判定规则：纯函数优先优化，不纯函数保证副作用执行顺序
3. 优化函数代码
   - 纯函数内的常量计算→编译期求值，避免运行期计算
   - 重复调用的纯函数→缓存结果
   - 判定规则：不改变函数行为的前提下最大化优化

### 13. 逃逸分析
1. 分析变量逃逸场景
   - 检测变量是否逃逸定义作用域（返回值/闭包捕获/跨函数赋值），建立「逃逸表」
   - 标记逃逸层级（函数内/模块内/全局）
   - 判定规则：未逃逸→栈分配，逃逸→堆分配
2. 推导内存分配策略
   - 未逃逸+固定大小→栈分配，未逃逸+动态大小→栈上`Vec`/`ArrayVec`
   - 逃逸+任意大小→堆分配（`Box`/`Rc`）
   - 判定规则：优先栈分配，逃逸时最小化堆分配（如`Box`而非`Rc`）
3. 优化逃逸代码
   - 消除不必要的逃逸（如将闭包捕获改为借用）
   - 逃逸小类型→`Copy`代替`Clone`
   - 判定规则：不改变逻辑的前提下减少逃逸

### 14. 拷贝/移动语义推导
1. 分析类型的`Copy`属性
   - 检测TS类型是否为基础类型（number/string/boolean）→对应Rust`Copy`类型
   - 标记复合类型的字段是否全部实现`Copy`
   - 判定规则：所有字段`Copy`→类型自动推导`Copy`，否则仅`Clone`
2. 推导拷贝/移动策略
   - `Copy`类型+短生命周期→直接拷贝，非`Copy`类型→优先移动
   - 多次使用的非`Copy`类型→自动插入`clone()`
   - 判定规则：避免不必要的`clone`，优先移动（零成本）
3. 生成拷贝/移动代码
   - `Copy`类型→直接赋值，非`Copy`类型→`move`赋值
   - 需共享的非`Copy`类型→`Rc::clone`（浅拷贝）
   - 判定规则：`clone`仅用于必要场景，优先`Copy`/`move`

### 15. 方法调度推导
1. 分析方法调用场景
   - 检测TS中的方法调用（`a.fn()`），标记调用类型（静态调用/动态调用）
   - 分析方法接收者类型（值/引用/可变引用）
   - 判定规则：静态类型已知→静态分发，类型动态→动态分发
2. 推导调度方式
   - 静态分发→`impl Trait`/直接调用，动态分发→`dyn Trait`+虚表
   - TS原型链方法→自动生成Rust`trait`+`impl`
   - 判定规则：优先静态分发（性能），动态分发仅用于类型不确定场景
3. 生成方法调度代码
   - 静态分发→`impl A { fn fn_(&self) {} }`，动态分发→`Box<dyn Fn()>`
   - 多态方法→自动生成`trait`和实现
   - 判定规则：保证方法调用行为与TS一致

### 16. 依赖 & 生命周期顺序推导
1. 分析结构体初始化顺序
   - 遍历TS类的构造函数，提取字段初始化顺序，建立「初始化依赖表」
   - 标记循环初始化依赖（A的初始化依赖B，B依赖A）
   - 判定规则：无循环→按顺序初始化，有循环→延迟初始化
2. 推导生命周期顺序
   - 确定字段的生命周期依赖（字段A的生命周期≤字段B）
   - 检测未初始化字段的使用（TS允许，Rust禁止）
   - 判定规则：使用前必须初始化，循环依赖→`Option<T>`+延迟赋值
3. 生成初始化代码
   - 无循环→直接结构体初始化（`A { b: B::new() }`）
   - 有循环→`Option<T>`+后续赋值（`let mut a = A { b: None }; a.b = Some(b);`）
   - 判定规则：避免未初始化字段，循环依赖优先`Option`而非`unsafe`
4. 入口函数（main）自动生成
   - 分析 TS 代码的执行入口：提取所有「全局作用域代码」（函数定义外的代码，如 `a=100; fn();`），作为 Rust `main` 函数的核心逻辑；
   - 调整函数/变量的定义顺序：将 TS 中「先函数后变量」的定义，重构为 Rust 合法的「先变量后函数」（嵌套在 `main` 内），或把函数提升为模块级函数（需处理变量捕获）；
   - 处理模块级函数的变量依赖：如果 TS 全局函数依赖全局变量，转换器自动将全局变量作为参数传入 Rust 函数，或包装为 `static` 变量；
   - 判定规则：
     ① TS 全局作用域的可执行代码（赋值、函数调用、循环等）→ 全部放入 Rust `main` 函数；
     ② TS 全局函数定义 → 优先嵌套在 `main` 内（方便访问变量），复杂函数提升为模块级函数；
     ③ 全局变量 → 非并发场景→`main` 内的局部变量，并发场景→`static`（需加 `Sync`/`Mutex`）。
