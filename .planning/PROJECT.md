# Transmute - TypeScript 到 Rust 语义转换器

## 项目愿景

构建一个高性能、准确的 TypeScript 到 Rust 源代码转换工具，能够自动推导所有权、生命周期等 Rust 语义特性。

## 项目目标

### 功能目标
1. **完整语法支持** - 支持 TypeScript 核心语法和常用特性
2. **语义准确转换** - 准确理解 TypeScript 语义并转换为等价的 Rust 代码
3. **自动所有权推导** - 自动分析并标注所有权、生命周期
4. **多后端解析** - 支持多种 TypeScript 解析器（Deno、swc 等）

### 质量目标
1. **测试覆盖** - 测试覆盖率 >= 80%
2. **正确率** - 对于标准测试用例，编译通过率 >= 95%
3. **性能** - 单个文件转换时间 < 0.5 秒
4. **内存安全** - 所有生成的 Rust 代码必须通过 Rust 编译器的安全检查

## 架构设计

### 5层架构

```
TypeScript 源代码
    ↓
[层1: 基础解析层 parser/] - 解析为标准化 JSON AST
    ↓
[层2: 语义分析层 semantic/] - 构建 IR（作用域、符号表、类型推断、控制流）
    ↓
[层3: 语义重构核心层 refactor/] - 15个自动推导算法（所有权、生命周期等）
    ↓
[层4: Rust AST层 ast/] - Rust 语义表示
    ↓
[层5: 代码生成层 codegen/] - 渲染为 Rust 源代码
    ↓
Rust 源代码
```

### 各层职责

#### 层1: 基础解析层 (parser/)
- 将 TypeScript 源代码解析为标准化的 JSON AST
- 支持多后端（Deno, swc）
- 统一 AST 表示

#### 层2: 语义分析层 (semantic/)
- 作用域分析
- 符号表构建
- 类型推断
- 控制流分析

#### 层3: 语义重构核心层 (refactor/)
- 15个全自动推导算法（并行执行）
- 所有权标注
- 生命周期推导

#### 层4: Rust AST层 (ast/)
- Rust 语义表示
- 独立于代码生成

#### 层5: 代码生成层 (codegen/)
- AST 到 Rust 源代码渲染
- 代码优化

## 15个推导算法

1. **所有权推导** - 确定值的所有权转移、借用关系
2. **生命周期推导** - 推导引用生命周期约束
3. **可变性推导** - 确定变量是否需要 mut
4. **引用类型推导** - 选择 &T / &mut T / Box<T> / Rc<T> / Arc<T>
5. **闭包获取模式推导** - 确定 Fn/FnMut/FnOnce 和 move 关键字
6. **空安全推导** - 转换 T | null 为 Option<T>
7. **类型精度推导** - 选择 i8/i16/i32/i64/u8/u16/u32/u64
8. **线程安全推导** - 推导 Send/Sync trait bounds
9. **内存分布推导** - 决定栈/堆分配策略
10. **异步模型推导** - 转换 async/await 为 Future
11. **函数纯度推导** - 识别纯函数，考虑 const 标记
12. **逃逸分析推导** - 分析值是否逃逸函数
13. **拷贝/移动推导** - 决定类型是否实现 Copy
14. **方法调度推导** - 选择静态/动态分发
15. **依赖/生命周期顺序推导** - 构建类型和生命周期依赖图
16. **错误处理推导** - 将 TS 的 throw/try/catch/Promise.reject 映射为 Rust 合规的错误处理逻辑
## 项目约束

### 技术约束
1. **Rust 版本** - 2024 Edition
2. **最低 Rust 版本** - 1.85.0
3. **JavaScript 运行时** - 需要 Node.js >= 20 或 Deno >= 2.0
4. **系统** - 支持 Windows、Linux、macOS

### 开发约束
1. **代码风格** - 使用 `rustfmt` 格式化，通过 `clippy` 检查
2. **测试覆盖** - >= 80%
3. **文档** - 所有公共 API 必须有文档注释
4. **错误处理** - 使用 `thiserror` 和 `anyhow` 进行统一错误处理

### 性能约束
1. **单个文件转换** - < 0.5 秒
2. **内存使用** - < 100MB
3. **并发处理** - 支持并行处理多个文件

## 开发原则

### 引用优先与性能 (CRITICAL)
- 优先使用引用而不是克隆对象
- 使用 `&T` 和 `&mut T` 避免所有权转移
- 仅在必须时才使用 `clone()`
- 使用迭代器避免中间分配

### 不可变性
- 使用不可变数据结构
- 避免内部可变性，除非有明确性能需求

### 文件组织
- 许多小文件 > 少数大文件
- 高内聚，低耦合
- 200-400 行典型，800 行最大

### 错误处理
- 在每个层面显式处理错误
- 提供用户界面的友好错误消息
- 记录服务器端的详细错误上下文
- 从不静默地吞咽错误

### 输入验证
- 在系统边界验证所有用户输入
- 使用基于模式的验证
- 快速失败并显示清晰的错误消息
- 不信任外部数据
