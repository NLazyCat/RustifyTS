# Phase 04: Semantic Refactoring Core Layer - Wave 2

**Plan ID:** 04-02
**Wave:** 2/3
**Estimated Time:** 3 days
**Dependencies:** Wave 1 complete (AnalysisContext and analyzer traits)

## Goal

Implement the Reference Relation Analysis (RRA) system and dataflow framework. RRA builds a graph of reference relationships between symbols, tracks usage patterns, and provides inter-procedural summaries for cross-function analysis.

## Context

From **04-CONTEXT.md** and **04-RESEARCH.md**, we have locked decisions:

**RRA Design:**
- Reference graph representation: Adjacency list - edge list (from_id, to_id, type)
- Reference type tracking: Typed edges (borrow, own, read, write as enum types)
- Analysis scope: Inter-procedural with summaries (cross-function call analysis with summary data)
- Reference usage tracking: Full usage tracking (track both usage counts and live ranges)

From research, we must:
- Use adjacency lists with Vec<Vec<Edge>> for sparse graphs (typical case)
- Implement lattice-based dataflow analysis framework
- Use hashbrown/fxhash for performance-critical collections
- Track both outgoing and incoming edges for bidirectional queries
- Store edge indices for O(1) lookup

## Must-Haves (Goal-Backward Verification)

To verify this wave succeeds, we MUST have:

1. **Reference graph with adjacency list** that:
   - Uses Vec<Vec<Edge>> for successors and predecessors
   - Stores edges with (target, ref_type) tuple

   - Provides successors() and predecessors() iterators
   - Uses NodeId (newtype) for type safety

2. **RefType enum** covering:
   - Ref (immutable borrow)
   - RefMut (mutable borrow)
   - Move (ownership transfer)
   - Read (read access)
   - Write (write access)

3. **UsageInfo tracking** with:
   - use_count (number of times symbol is used)
   - live_blocks (basic blocks where symbol is live)

4. **RRA analyzer** that:

5. **Inter-procedural summaries** with:
   - Parameter reference types (which are borrowed/moved)

6. **Dataflow framework** with:
   - Analysis trait (Domain, START_STATE, DIRECTION, apply)

   - Forward and backward direction support

7. **Tests** covering:
   - Graph construction and edge operations

## Notes

- Adjacency list is O(E) storage, O(1) edge iteration (optimal for sparse graphs)
- RefType enum enables precise reference tracking (borrow vs move)
- Dataflow analysis uses backward direction for liveness (propagate from uses)
- Lattice join is union for BitSet (live(A ∪ B) = live(A) ∪ live(B))
- Inter-procedural summaries are instance-based (type-based caching can be added later)
- max_iterations should be set to 100 (adjust based on empirical testing)
- FxHashMap used for performance (2-3x faster than std HashMap)
