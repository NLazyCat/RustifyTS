# Phase 04: Semantic Refactoring Core Layer - Wave 3

**Plan ID:** 04-03
**Wave:** 3/3
**Estimated Time:** 3 days
**Dependencies:** Wave 2 complete (RRA and dataflow framework)

## Goal

Implement Ownership Annotator (OA), Semantic Rewriter (SR), and derivation algorithms (ownership, lifetime, mutability). These components use RRA's reference analysis to infer and annotate ownership information, apply semantic transformations to the IR, and derive Rust-specific properties.

## Context

From **04-CONTEXT.md** and **04-RESEARCH.md**, we have locked decisions:

**OA Design:**
- Annotation storage: Inline in IR nodes (attached directly to IR nodes)
- Ownership determination: Mixed approach (automatic inference + manual hints)
- Conflict handling: Track ambiguity (mark as ambiguous for later resolution)
- Annotation format: Rust-style categories (Owned, Ref, RefMut)

**SR Design:**
- IR modification: Immutable copy (copy IR and apply transformations)
- Rewrite rule application: Fixed-point iteration (iterate until fixed point)
- Rewrite rule structure: Visitor-based (IR visitor pattern)
- Transformation tracking: Full history (track all transformations for debugging/rollback)

**Derivation Algorithms:**
- Algorithm order: Sequential pipeline (RRA → OA → derivation)
- Result storage: Context results map (stored in AnalysisContext results map)
- Incremental analysis: Full incremental (support incremental updates on IR changes)
- Cache strategy: Lazy cache (lazy caching, invalidate on IR changes)

From research, we must:
- Use visitor pattern for IR traversal
- Implement fixed-point iteration with max_iterations
- Store transformation history efficiently (diffs, not full IR copies)
- Use Rust-style ownership categories
- Support manual hints override

   - `BorrowInfo` struct: `borrower: ValueId`, `borrowed: ValueId`, `ref_type: RefType`
   - `LifetimeDeriver` implementing SemanticAnalyzer<LifetimeDerivation>
   - Derivation logic:
     - Use liveness from RRA
     - Compute borrow regions from CFG
     - Derive lifetime constraints from control flow
